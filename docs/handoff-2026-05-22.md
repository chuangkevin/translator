## 產品概述

Chrome MV3 擴充套件，透過自架的 OpenCode 伺服器（呼叫 AI 模型）將任意語言網頁翻譯成繁體中文。

核心功能：
- **雙語對照模式**：在原文段落下方插入中文譯文（非覆蓋）
- **簡體中文原地替換**：偵測到簡體就直接換掉原文，不插入雙語
- **YouTube 字幕翻譯**：預先抓取全段 VTT 並翻譯，用 `position: fixed` overlay 顯示
- **動態內容翻譯**：MutationObserver 監聽 DOM 新增節點，捲動載入的留言也會自動翻
- **右側懸浮 FAB**：可拖曳、吸附邊緣，顯示翻譯狀態
- **Alt+A**：快捷鍵切換雙語模式
- **網站規則**：可設定特定 domain 永遠翻 / 永遠不翻

## 架構

```
使用者操作 / 頁面載入
        │
[content.ts]                   [content-youtube.ts]
  BilingualInjector               YoutubeCaptionTranslator
  FloatingButton                    ├─ getCaptionUrlFromPageScript()
  MutationObserver                  ├─ 輪詢 dataset.xtCaptionUrl
  yt-navigate-finish listener       └─ fetchCaptionUrlFromApi() fallback
        │                                       │
        └──────────── sendMessage ──────────────┘
                              │
                     [background.ts]
                       OpenCodeClient
                       Translator (Semaphore 5)
                              │
                    POST /session
                    POST /session/{id}/message
                    DELETE /session/{id}
                              │
                    [OpenCode Server]
                    (自架，呼叫 AI 模型)

[background.ts] 另外負責：
  - chrome.tabs.onUpdated → executeScript(ytCaptionBridge, world: MAIN)
    將 ytInitialPlayerResponse 的字幕 URL 寫入 dataset.xtCaptionUrl
  - chrome.commands.onCommand → relay Alt+A 到 active tab
```

### 翻譯流程（網頁）

1. `isTraditionalChinesePage()` 判斷頁面語言（YouTube 特例：取樣標題文字而非 `lang` 屬性）
2. 非繁中 → `toggleBilingual()` → `startObserver()` + `translatePage()`
3. `getTargets()` 取出 `p, h1~h6, li, td, blockquote, #content-text, #video-title`
4. `Promise.all` 並行翻譯所有元素，每個各開一個 OpenCode session
5. 簡體中文 → `replaceSimplified()`（存原文到 `data-xt-orig`）；其他 → 插入 `.xt-translation` 節點
6. 翻譯期間與翻譯後，MutationObserver 補抓動態新增節點（debounce 200ms）

### 字幕翻譯流程（YouTube）

1. `content-youtube.ts` 啟動 `YoutubeCaptionTranslator`
2. `waitForCaptionUrl()` 依序嘗試：①解析 `<script>` 標籤（最快）→ ②`dataset.xtCaptionUrl`（background 注入）→ ③`/api/timedtext` API
3. fetch VTT → `parseVtt()`（去 HTML tag、解 entity、去重）
4. 5 個 concurrent workers 翻譯所有字幕段
5. RAF loop 根據 `video.currentTime` 對應段落，更新 `position: fixed` overlay 文字

## 技術注意事項

### OpenCode Session 模式
每次翻譯都是獨立的 session lifecycle：`POST /session` → `POST /session/{id}/message` → `DELETE /session`。無狀態，每個請求耗時 2–10 秒。並行請求沒有全域限流（background 每個 message 都 `new Translator()`，Semaphore 實際無效）。

### YouTube CSP
YouTube 的 CSP 禁止從 content script 注入 `<script>` 標籤。繞過方式：background 用 `chrome.scripting.executeScript({ world: 'MAIN' })` 注入 `ytCaptionBridge()`，讀取 `ytInitialPlayerResponse` 後寫進 `dataset.xtCaptionUrl`（DOM attribute，isolated world 可讀）。

### 語言偵測
- `lang` 屬性以 `zh-TW/zh-HK/zh-Hant` 開頭 → 繁中，不翻
- YouTube 特例：`lang="zh-TW"` 反映使用者 UI 語言，非影片語言，改取樣標題元素文字判斷
- `lang` 為空或 `zh`：取樣 body 前 500 字判斷是否為簡體（呼叫 `isSimplifiedChinese()`）

### 簡體中文偵測
`isSimplifiedChinese()` 使用固定字元集（`们书爱见说...`）比對。偵測到日文假名（hiragana/katakana）時提前返回 false，避免誤判日文。

### MutationObserver 防重複
- 正在翻譯的元素：設 `data-xt-id` → `getTargets()` / `getNewTargets()` 過濾掉
- 已翻譯（簡體替換）：設 `data-xt-orig` → 同上過濾
- 已翻譯（雙語注入）：有 `.xt-translation` class 的 sibling → `getTargets()` 不選它
