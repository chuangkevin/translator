## 架構快覽

```
[content.ts]         ──sendMessage─→  [background.ts]
[content-youtube.ts] ──sendMessage─→  [background.ts]
                                             │
                                    POST /session
                                    POST /session/{id}/message
                                    DELETE /session/{id}
                                             │
                                    [OpenCode Server]
                                    provider: openai
                                    model: chatgpt5.5
```

- **所有 HTTP 請求**集中在 Background SW，content script 不直接打外部 API。
- **設定**存在 `chrome.storage.sync`（serverUrl / provider / model / targetLang / bilingualEnabled / selectionEnabled）。
- **OpenCode Client** 無 auth token（不送 Authorization header）。

## 已知品質修正記錄

### Task 6 bilingual-injector

原始計畫版本有兩個 bug，已修正並補測試：

1. **`idCounter` 從 module-level 改為 instance field**，`clear()` 時重設為 0，避免跨測試 / 跨實例污染。
2. **`inject()` 改用 `this.ownerDoc` getter** 取代硬寫 `document.createElement`，確保在非 global document（如 iframe）下也能正確建立 DOM 節點。
3. **補 round-trip 測試**：inject → clear → re-inject，確認清除後狀態完全還原。

### Task 7 selection-popup（品質問題已知但未修正）

Quality reviewer 發現兩個 Important 等級問題，尚未修正：

1. **`escapeHtml` 未 escape 單引號**（`'` → `&#39;`）。目前文字插入在 `<div>` 內文，暫無直接 XSS 風險，但若未來用於 HTML 屬性則有隱患。
2. **選取翻譯無 request-ID 防護**：使用者快速選取第二段文字時，第一次翻譯結果可能覆蓋第二次的 popup 顯示。

這兩個問題未在原始計畫 spec 內，可在下一個對話中修正（加 `&#39;`、加 `requestId` instance field）。
